Name

vfs_default_quota — Windowsクライアント用の容量制限(quota)レコードを保存する

Synopsis

vfs objects = default_quota

説明

このVFSモジュールは samba(7)システムの一部である.

多くの一般的な容量制限の実装はユーザやグループ単位で設定することはできるが、 既定値を指定することができない。 Sambaのvfs_default_quotaモジュールは容量制限の 初期値を持つことができ、Windowsのエクスプローラーから調べることができる。

初期設定では、SambaはLO_LIMITをデフォルトクオータとして返却し、 内容は変更できない。 vfs_default_quotaはデフォルトクオータをユーザの クオータレコードの内容に読み替えて返却する。 初期設定では、ルートユーザはこの動作をしない。なぜなら容量制限は通常 ルートユーザに有効にならないからである。

このモジュールはスタッカブルである。

オプション

default_quota:uid = UID

デフォルトユーザクオータが保存されているユーザIDを指定する。

default_quota:gid = GID

デフォルトユーザクオータが保存されているグループIDを指定する

default_quota:uid nolimit = BOOL

パラメータにTrueを指定すると、デフォルトユーザクオーターが指定されていても 容量制限なし(NO_LIMIT)と通知される。そうでない場合、保存されている値がレポートされる。

default_quota:gid nolimit = BOOL

パラメータにTrueを指定すると、デフォルトグループクオーターが指定されていても 容量制限なし(NO_LIMIT)と通知される。そうでない場合、保存されている値がレポートされる。

使用例

ユーザーID65535のクオータレコードにデフォルトクオータレコードを保存する。 そして、そのユーザはクオータ制限なしとする。 

        [global]
	vfs objects = default_quota
	default_quota:uid = 65535
	default_quota:uid nolimit = yes

バージョン

このマニュアルはSambaバージョン3.0.25に対応している。
